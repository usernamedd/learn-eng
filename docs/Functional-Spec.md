# 功能规格说明书

## 概述

learn-eng 是一个跨平台英语学习助手，提供**翻译练习**和**对话练习**两种学习模式。

本文档基于 [`PRD.md`](PRD.md) 进行细化，描述每个功能的详细行为、交互流程和界面要求。

---

| 编号 | 功能 | 优先级 | 所属上下文 |
|------|------|--------|-----------|
| F1 | 翻译模式 | P0 | Translation |
| F2 | 对话模式 | P0 | Conversation |
| F3 | 语言切换 | P0 | Configuration |
| F4 | 模型配置 | P1 | Configuration |
| F5 | 配置管理界面 | P1 | Configuration |
| F6 | 语音识别（STT） | P1 | Conversation |
| F7 | 语音合成（TTS） | P1 | Conversation |

## 二、功能详述

### F1 翻译模式

**描述：** 用户说一句中文，程序翻译成目标语言并以文字形式展示。

**用户流程：**

```
① 用户点击语音输入按钮，说出中文句子
② 程序识别语音（F6），转为文字
③ 调用 LLM 翻译成目标语言
④ 展示翻译结果（源文字 + 译文对照）
```

**界面要素：**

| 元素 | 说明 |
|------|------|
| 语音输入按钮 | 触发录音，录制时显示波形/倒计时 |
| 源文字区域 | 显示用户语音识别后的中文原文 |
| 翻译结果区域 | 显示目标语言的翻译文字 |
| 当前语言标识 | 显示正在使用的目标语言（如 "🇺🇸 EN"） |

**交互细节：**

- 翻译完成后自动停止录音
- 翻译结果显示后，用户可点击播放按钮朗读译文（TTS）
- 用户可手动输入文字替代语音输入

**验收标准：**

- [ ] 用户语音输入中文，程序正确识别并显示原文
- [ ] 翻译结果在 5 秒内返回
- [ ] 翻译结果以文字形式清晰展示
- [ ] 支持手动输入文字进行翻译
- [ ] 翻译过程中显示加载状态

---

### F2 对话模式

**描述：** 用户用中文说话，AI 用目标语言回复，同时显示文字并朗读语音。

**用户流程：**

```
① 用户点击语音输入按钮，说出中文句子
② 程序识别语音（F6），转为文字
③ 调用 LLM，AI 以目标语言回复
④ 展示 AI 回复文字 + 自动语音朗读（F7）
⑤ 用户继续对话，对话历史保留上下文
```

**界面要素：**

| 元素 | 说明 |
|------|------|
| 对话消息列表 | user 消息（中文）和 assistant 消息（目标语言）交替展示 |
| 语音输入按钮 | 触发录音 |
| 语音播放按钮 | 每条 AI 回复旁带播放按钮，可重新朗读 |
| 清空对话按钮 | 清空当前对话历史，开始新对话 |
| 当前语言标识 | 显示正在使用的目标语言 |

**交互细节：**

- 对话历史在会话期间保留，AI 能理解上下文
- 用户可随时点击「清空」重置对话
- AI 回复显示后自动触发 TTS 朗读
- AI 回复中若用户出现语法错误，可温和纠正

**验收标准：**

- [ ] 用户语音输入后，AI 在 5 秒内用目标语言回复
- [ ] AI 回复自动朗读语音
- [ ] 对话历史保持上下文（连续 5 轮以上对话无歧义）
- [ ] 清空对话后重新开始，之前的历史不影响新对话
- [ ] 每条 AI 回复可手动触发重新朗读

---

### F3 语言切换

**描述：** 用户可随时切换目标语言，切换后翻译模式和对话模式均立即生效。

**支持语言：**

| 语言 | 代码 | Emoji |
|------|------|-------|
| English | en | 🇺🇸 |
| 日本語 | ja | 🇯🇵 |
| 한국어 | ko | 🇰🇷 |
| Deutsch | de | 🇩🇪 |
| Français | fr | 🇫🇷 |
| Español | es | 🇪🇸 |
| 中文（源语言） | zh | 🇨🇳 |

**用户流程：**

```
① 用户在设置/快捷切换入口选择目标语言
② 配置持久化保存
③ 翻译模式和对话模式的下一次请求使用新语言
```

**交互细节：**

- 主界面显示当前目标语言标识（Emoji + 缩写）
- 点击语言标识可弹出语言选择列表
- 语言切换**不中断**当前对话（新的回复才使用新语言）

**验收标准：**

- [ ] 切换语言后，翻译模式的输出使用新语言
- [ ] 切换语言后，对话模式的 AI 回复使用新语言
- [ ] 语言选择持久化，下次启动应用保留上次选择
- [ ] 切换操作不超过 2 步

---

### F4 模型配置

**描述：** 用户可选择 AI 服务提供商并配置 API Key。

**支持提供商：**

| 提供商 | 默认模型 | 类型 |
|--------|---------|------|
| DeepSeek | deepseek-chat | 云端 |
| OpenAI | gpt-4 | 云端 |
| Claude | claude-3 | 云端 |
| Gemini | gemini-pro | 云端 |
| Ollama | llama3 | 本地部署 |
| vLLM | — | 本地兼容 |

**配置项：**

| 配置 | 说明 | 默认值 |
|------|------|--------|
| LLM 提供商 | 选择 AI 服务商 | deepseek |
| API Key | 对应提供商的认证密钥 | 空 |
| Base URL | 自部署模型的地址（Ollama/vLLM） | — |

**验收标准：**

- [ ] 用户可选择任意支持的 AI 提供商
- [ ] 配置修改后，下一次请求使用新配置
- [ ] API Key 以密文形式存储
- [ ] 切换提供商后，对话历史不受影响

---

### F5 配置管理界面

**描述：** 应用内提供配置页面，用户可查看和修改所有配置。

**界面要素：**

| 元素 | 说明 |
|------|------|
| 目标语言选择 | 下拉列表选择目标语言 |
| LLM 提供商选择 | 下拉列表选择 AI 服务商 |
| API Key 输入 | 按提供商分别配置 API Key（密码输入框） |
| Base URL 输入 | 自部署模型的地址输入（Ollama/vLLM 时显示） |
| 保存按钮 | 持久化配置 |

**验收标准：**

- [ ] 配置页面可进入并显示当前所有配置
- [ ] 修改后保存，配置持久化
- [ ] 回到主界面后，新配置生效

---

### F6 语音识别（STT）

**描述：** 将用户语音输入转为文字。

**技术依赖：**

- 桌面端：使用 Web Speech API / 系统麦克风
- 移动端：使用平台原生 STT（Android SpeechRecognizer / iOS SFSpeechRecognizer）

**行为：**

- 用户按住/点击语音按钮开始录音
- 松开/再次点击停止录音
- 音频发送至 STT 引擎识别
- 识别结果以文字形式返回

**验收标准：**

- [ ] 中文语音识别准确率 > 90%
- [ ] 录音过程中有 UI 反馈（波形/电平指示）
- [ ] 识别结果在 3 秒内返回

---

### F7 语音合成（TTS）

**描述：** 将 AI 回复的文字用目标语言朗读出来。

**技术依赖：**

- 桌面端：使用 Web Speech API / 系统 TTS
- 移动端：使用平台原生 TTS（Android TextToSpeech / iOS AVSpeechSynthesizer）

**行为：**

- AI 回复完成后自动触发 TTS
- 用户可手动点击播放按钮重新朗读
- 使用目标语言的发音（如日文用日文发音，英文用英文发音）

**验收标准：**

- [ ] AI 回复后自动朗读语音
- [ ] 手动点击可重新播放
- [ ] 朗读过程中可打断并重新开始
- [ ] 发音为目标语言标准发音

## 三、模式对比

| 维度 | 翻译模式 | 对话模式 |
|------|---------|---------|
| 输入 | 中文语音/文字 | 中文语音/文字 |
| 输出 | 目标语言翻译文字 | 目标语言 AI 回复（文字 + 语音） |
| 会话状态 | 无状态，单次翻译 | 有状态，保留对话历史 |
| AI 角色 | 翻译引擎 | 语言陪练 |
| 适用场景 | 查单词、确认表达 | 练习口语对话 |
